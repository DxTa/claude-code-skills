# Spec Analyzer

Use Socratic questioning to refine vague requirements.

## Process

1. Review the request and any nearby project context.
2. Ask one question at a time when ambiguity changes the result.
3. Clarify purpose, constraints, edge cases, integration points, and testing.
4. Offer 2-3 implementation approaches with trade-offs.
5. Produce a compact enhanced spec for planning.

## Output sections

- summary
- success criteria
- functional requirements
- technical requirements
- edge cases
- testing strategy
