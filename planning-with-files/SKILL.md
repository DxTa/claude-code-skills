# Planning With Files

Use markdown files as durable working memory.

## File locations

- `docs/plans/<topic>.md` for implementation plans
- `docs/plans/<topic>-notes.md` for research notes when needed

## Expectations

1. Keep a clear goal statement.
2. Break work into phases.
3. Record decisions and errors.
4. Update status as you move between phases.
5. Record delegation only when subagents are actually used.

## Recommended template

```markdown
# Task Plan: <topic>

## Goal

## Phases
- [ ] Plan
- [ ] Build
- [ ] Verify

## Decisions Made

## Delegation Summary
Include only when subagents are dispatched.

## Errors Encountered

## Status
```
