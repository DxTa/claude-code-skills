# Brainstorming

Start by understanding the user's goal and the current project context.

## Process

1. Ask one clarifying question at a time when a decision materially changes the design.
2. Propose 2-3 approaches with trade-offs.
3. Lead with the recommended option and explain why.
4. Present the design in small sections and validate it incrementally.

## Design coverage

- architecture
- components
- data flow
- error handling
- testing

## Documentation

Save validated designs to `docs/plans/YYYY-MM-DD-<topic>-design.md`.
